# Dalfox: Beginner's Guide for Kali Linux

Dalfox is a powerful tool for finding Cross-Site Scripting (XSS) vulnerabilities. This guide will help you get started on Kali Linux.

## 1. Installation

Kali Linux includes Dalfox in its official repositories, making it very easy to install.

```bash
sudo apt update
sudo apt install dalfox -y
```

Alternatively, if you want the very latest version, you can install it via Go:

```bash
go install github.com/hahwul/dalfox/v2@latest
```

## 2. Basic Usage

### Scan a Single URL
The most common way to use Dalfox is to scan a single URL with parameters.

```bash
dalfox url "http://testphp.vulnweb.com/listproducts.php?cat=1"
```

### Scan Multiple URLs from a File
If you have a list of targets in a text file (e.g., `targets.txt`), use the `file` mode.

```bash
dalfox file targets.txt
```

### Piping from Other Tools
Dalfox works great in a pipeline with other tools like `subfinder` or `httpx`.

```bash
# Example: Find subdomains, check for live hosts, then scan for XSS
cat subdomains.txt | httpx | dalfox pipe
```

## 3. Essential Flags

| Flag | Description | Example |
| :--- | :--- | :--- |
| `-b`, `--blind` | Add a Blind XSS callback URL | `-b yourname.xss.ht` |
| `-H`, `--header` | Add custom HTTP headers (e.g., Cookies) | `-H "Cookie: session=123"` |
| `--json` | Output results in JSON format (best for logs) | `--json` |
| `--silence` | Suppress the banner and info logs | `--silence` |
| `--proxy` | Send requests through a proxy (like Burp Suite) | `--proxy http://127.0.0.1:8080` |

## 4. Understanding the Results

When Dalfox finds something, it will highlight it in the console:

*   **`[V] Vulnerable`**: A confirmed XSS vulnerability was found.
*   **`[W] Weak`**: A potential issue was found, but it might be partially filtered or mitigated.
*   **`[I] Info`**: General information about the page's security headers or parameters.

## 5. Pro Tip: Automation
For a fast and clean scan that only shows you the vulnerabilities found in JSON format:

```bash
dalfox url "http://example.com/page.php?id=1" --silence --json
```
